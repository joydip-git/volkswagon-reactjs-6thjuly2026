sharing data between components
	a. passing data from parent to a immediately or shallowly nested component and vice versa
		- pass data via props
	b. sharing data between deeply nested component and a parent component
		- from parent save data in a context object which can be accessed by any child in the same tree
	c. sharing data between any components whether they are in the same or different tree

Context Object: used to share data between components in the same tree
	1. to create -> const SomeContext = createContext<T>(initialData)
	2. to provide data from a parent -> 
		<SomeContext.Provider value={of type T}>
			<the component tree>
		</...>
	3. to consume data from a child in the same tree
		const contextData = useContext(SomeContext)
Note: no component outside the scope of the component tree will be able to get latest data from context.

Redux terminologies:
-------------------------
1. state => an object with properties and values which is part of the redux store and is updated by reducer function. any component can subscribe to this state. whenever the state is updated the subscriber will get the updated state data from the store and the component will be re-rendered

2. reducer => a function (which as two arguments - first: reference to the previous state, second: an action object). a reducer function is called by store and the reducer function must return a new state to the store. Since for the very first time store does not have any state reference that's why the first argument of reducer function must be assigned to an initial state. Next time onwards while calling the reducer function the store will pass the reference of the previous state (as returned by the reducer function in the previous step).
the reducer function MUST be used to create a new state based on the previous state. the reducer function replaced the previous state by the new state

3. store => an object which is collection  of reducer function(s) (thus indirectly collection of state(s)). store recives an action from any component and invokes the reducer method and passes the copy of the previous state object and the received action object as parameters to it. after reducer returns the new state object to the store, it notifies every component subscribed to the state and react re-renders those components

4. action => an object which is dispatched against a redux store. It must have at least one property named, type along with any other additional property

5. dispatch => a function of store which carries an action object to the store

6. reducer map => an object (map of reducers) which contains properties, where each of them referes to one reducer (thus one state indirectly)

useSelector hook from react-redux: A hook to access the redux store's state. This hook takes a selector function as an argument. The selector is called with the store state.

creating typed useSelector:
-----------------------------
useSelector.withTypes<T>() => Creates a "pre-typed" version of useSelector where the state type is predefined.

This allows you to set the state type once, eliminating the need to specify it with every useSelector call.


creating typed useDispatch:
----------------------------
useDispatch.withTypes<T>() => Creates a "pre-typed" version of useDispatch where the type of the dispatch function is predefined. (what type of action object should be passed to dispatch method as an argument)

This allows you to set the dispatch type once, eliminating the need to specify it with every useDispatch call.
